# Bangumi 推荐系统 — 整体架构与数据流

## 一、系统定位

基于 [Bangumi（番组计划）](https://bgm.tv) 的多引擎混合推荐系统。根据用户收藏数据和历史评分，自动选择合适的推荐引擎组合，加权融合后输出个性化动画推荐列表。

## 三、核心数据流（一次完整推荐过程）

```
用户请求
  │  CLI: python main.py recommend --user xxx
  │  API: GET /recommend?user_id=xxx
  ▼
┌──────────────────────────────────────────────────────────────────┐
│ 1. 检查用户存在                                                   │
│    api_client.check_user_exists(username) → Bangumi API         │
└──────────────────────────────────────────────────────────────────┘
  ▼
┌──────────────────────────────────────────────────────────────────┐
│ 2. 获取收藏数据                                                   │
│    fetch_watched_and_wished(username)                            │
│    ├── 优先读 api_cache.db (TTL 168h)                            │
│    └── 否则调 Bangumi API: /v0/users/{username}/collections      │
│        ├── collect_type=2 → watched   (看过)                     │
│        └── collect_type=1 → wished    (想看)                     │
│    → 同步到 user_interactions.db (sync_db)                       │
└──────────────────────────────────────────────────────────────────┘
  ▼
┌──────────────────────────────────────────────────────────────────┐
│ 3. 构建推荐上下文 (_build_context)                                │
│    ├── 从 user_interactions.db 加载:                              │
│    │   ├── watched_subject_ids  (看过列表)                        │
│    │   ├── wished_subject_ids   (想看集合)                        │
│    │   └── history_ratings      {subject_id: rate} 评分字典      │
│    ├── 通过 ID 映射判断是否在 GCN 训练数据中                       │
│    └── 构造 GCN 排除/热身列表                                     │
│    → 输出 RecommendContext 对象                                   │
└──────────────────────────────────────────────────────────────────┘
  ▼
┌──────────────────────────────────────────────────────────────────┐
│ 4. 推荐缓存检查 (可选)                                            │
│    recommend_cache.db → 命中则直接返回 JSON                       │
│    缓存 key: user_id + engine_combo + top_k                      │
└──────────────────────────────────────────────────────────────────┘
  ▼
┌──────────────────────────────────────────────────────────────────┐
│ 5. 引擎路由 (RecommendRouter)                                     │
│    根据用户画像选择引擎组合:                                        │
│                                                                   │
│    收藏 ≤ 50 部 → Content + Hot + LLM (small_collection)         │
│    GCN 训练用户  → GCN + LLM (gcn_user)                          │
│    其他          → ItemCF + Content + LLM (rich_no_gcn)          │
└──────────────────────────────────────────────────────────────────┘
  ▼
┌──────────────────────────────────────────────────────────────────┐
│ 6. 多引擎并行推荐                                                 │
│                                                                   │
│    ┌─────────┬─────────┬─────────┬─────────┬─────────┐          │
│    │  GCN    │ ItemCF  │ Content │  Hot    │  LLM    │          │
│    │ (图)    │ (协同)  │ (内容)  │ (热度)  │ (语义)  │          │
│    │         │         │         │         │         │          │
│    │LightGCN │物品协同  │staff/VA │热度加权  │overall_ │          │
│    │图卷积   │过滤     │/tag向量 │时间衰减  │comment  │          │
│    │Warm-    │评分偏差  │         │         │嵌入     │          │
│    │start    │         │         │         │         │          │
│    └────┬────┴────┬────┴────┬────┴────┬────┴────┬────┘          │
│         │         │         │         │         │                │
│         输出: Dict[EngineName, List[RecommendItem]]               │
│         各引擎独立返回 top_k 结果                                  │
└──────────────────────────────────────────────────────────────────┘
  ▼
┌──────────────────────────────────────────────────────────────────┐
│ 7. 加权融合 (WeightedFusion)                                      │
│                                                                   │
│    对每路结果:                                                     │
│    ├── Min-Max 归一化 (各引擎内分数 → [0, 1])                     │
│    ├── 乘以引擎权重 (profile → weights)                           │
│    ├── 合并: 同一 subject_id 取最高加权分                         │
│    ├── 排序: 降序取 top_k                                         │
│    └── 标记 is_wished, 分配 rank                                  │
│                                                                   │
│    融合权重示例:                                                   │
│    ┌─────────────────┬───────┬───────┬────────┬──────┐          │
│    │ Profile         │ GCN   │ ItemCF│ Content│ Hot  │ LLM     │
│    ├─────────────────┼───────┼───────┼────────┼──────┼─────────┤
│    │ small_collection│  -    │   -   │  0.30  │ 0.40 │ 0.30    │
│    │ gcn_user        │ 0.60  │   -   │   -    │  -   │ 0.40    │
│    │ rich_no_gcn     │  -    │ 0.40  │  0.30  │  -   │ 0.30    │
│    └─────────────────┴───────┴───────┴────────┴──────┴─────────┘│
└──────────────────────────────────────────────────────────────────┘
  ▼
┌──────────────────────────────────────────────────────────────────┐
│ 8. 结果缓存 & 输出                                                │
│    写入 recommend_cache.db → 输出 JSON / CLI 表格                │
└──────────────────────────────────────────────────────────────────┘
```

## 四、输入输出规格

### 输入

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `user_id` / `username` | `str` | 是 | Bangumi 用户 ID |
| `top_k` | `int` | 否 | 推荐数量，默认 100，范围 1-100 |
| `use_cache` | `bool` | 否 | 是否使用推荐缓存，默认 True |
| `force_refresh_api` | `bool` | 否 | 是否强制刷新 API 缓存 |
| `token` | `str` | 否 | Bangumi API Access Token |

### 输出

```json
{
  "user_id": "xxx",
  "source": "gcn+llm",
  "collection_count": 156,
  "wished_count": 23,
  "in_training_data": true,
  "gcn_exclude_items": 156,
  "gcn_new_for_warm": 12,
  "recommendations": [
    {
      "rank": 1,
      "subject_id": 12345,
      "name": "進撃の巨人",
      "name_cn": "进击的巨人",
      "score": 0.8521,
      "source": "gcn",
      "is_wished": false,
      "rating_score": 8.5,
      "reasons": ["semantic similarity: 0.8521"]
    }
  ]
}
```

## 五、数据库汇总

| 数据库 | 路径 | 用途 | 关键表 |
|--------|------|------|--------|
| `user_interactions.db` | `db/` | 用户收藏与评分 | `user_collections` |
| `anime_meta.db` | `db/` | 动画元数据 | `anime_entries` (28 字段) |
| `api_cache.db` | `db/` | API 响应缓存 | `collection_cache` |
| `recommend_cache.db` | `db/` | 推荐结果缓存 | `recommend_cache` |
| `seasonal.db` | `db/` | 季度动画数据 | `seasonal_anime`, `update_log` |

## 六、关键技术栈

| 组件 | 技术 |
|------|------|
| 入口 | Python CLI (argparse) + FastAPI |
| GCN | PyTorch + LightGCN (自实现) |
| ItemCF | JSON 预计算相似度矩阵 |
| Content | PyTorch + BGE-small-zh-v1.5 (预计算 pt) |
| LLM | NumPy + BGE-small-zh-v1.5 (预计算 npy) |
| Hot | SQLite 聚合 + 时间衰减 |
| 存储 | SQLite × 5 |
| 外部 API | Bangumi API (api.bgm.tv) |
